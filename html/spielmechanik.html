1 Einleitung

Die Spielmechanik von Wordle basiert auf einem intelligenten Algorithmus zur Übereinstimmungsprüfung, der in der Klasse CheckAlgo implementiert ist. Diese Klasse ist das Herzstück des Spiels und entscheidet, wie gut die Eingabe des Spielers mit dem gesuchten Wort übereinstimmt.

Der Algorithmus arbeitet in zwei Phasen: Zuerst werden exakte Übereinstimmungen (richtige Buchstaben an richtiger Position) gefunden und grün markiert. Anschließend werden Buchstaben identifiziert, die im Zielwort vorkommen, aber an der falschen Position stehen - diese werden gelb markiert. Alle anderen Buchstaben bleiben grau.

2 Die CheckAlgo-Klasse

Die CheckAlgo-Klasse wurde von Lennart und Moritz entwickelt und enthält die zentrale Methode checkWord(). Diese Methode nimmt zwei Parameter entgegen: das eingegebene Wort und das gesuchte Zielwort. Als Rückgabe erhält man ein Array von JavaFX-Farben, das für jede Buchstabenposition die entsprechende Farbe angibt.

Methodensignatur

Methoden-Deklaration

public static Color[] checkWord(String input, String targetWord)

Schritt 1: Eingabe vorbereiten

Zuerst werden beide Wörter in Großbuchstaben umgewandelt, um eine einheitliche Vergleichsbasis zu schaffen. Anschließend wird die Länge des Zielworts ermittelt und die benötigten Arrays erstellt.

Eingabe normalisieren und Arrays erstellen

// Konvertierung der Eingabe und des Zielwortes in Großbuchstaben
input = input.toUpperCase();
targetWord = targetWord.toUpperCase();

// Speicherung der Länge des Zielwortes
int wordLength = targetWord.length();

// Array, das die Farben für die Übereinstimmungen speichert
Color[] colors = new Color[wordLength];

// Hilfsarray, um zu verfolgen, welche Buchstaben im Zielwort bereits verwendet wurden
boolean[] usedInTargetWord = new boolean[wordLength];

Wichtige Arrays

colors[]: Speichert für jede Position die Hintergrundfarbe
usedInTargetWord[]: Verhindert, dass ein Buchstabe mehrfach als "gelb" markiert wird

3 Übereinstimmungsprüfung in zwei Phasen

Der Algorithmus arbeitet systematisch in zwei Durchgängen, um korrekte Ergebnisse zu gewährleisten und Dopplungen zu vermeiden.

Erster Durchgang: Exakte Übereinstimmungen finden

Im ersten Durchlauf werden alle Positionen überprüft, an denen der Buchstabe der Eingabe exakt mit dem Buchstaben des Zielworts übereinstimmt. Diese werden sofort grün markiert und als "verwendet" gekennzeichnet.

Erster Durchgang - Grüne Markierungen

// Erster Durchgang: Finden von exakten Übereinstimmungen
for (int i = 0; i < wordLength; i++) {
    if (i < input.length() && input.charAt(i) == targetWord.charAt(i)) {
        colors[i] = Color.web("#6aaa64"); // Grün für richtige Position
        usedInTargetWord[i] = true;
    } else {
        colors[i] = Color.web("#787c7e"); // Grau als Standardfarbe
    }
}

Zweiter Durchgang: Buchstaben an falscher Position

Im zweiten Durchlauf werden alle Positionen betrachtet, die noch nicht grün markiert sind. Für jeden Buchstaben wird geprüft, ob er irgendwo anders im Zielwort vorkommt und noch nicht verwendet wurde.

Zweiter Durchgang - Gelbe Markierungen

// Zweiter Durchgang: Finden von Buchstaben an falscher Position (gelb)
for (int i = 0; i < input.length() && i < wordLength; i++) {
    if (colors[i].equals(Color.web("#6aaa64"))) {
        continue; // Bereits als korrekt markiert, überspringen
    }

    char letters = input.charAt(i);
    boolean lettersFound = false;

    // Suche nach diesem Buchstaben im Zielwort
    for (int j = 0; j < wordLength; j++) {
        if (!usedInTargetWord[j] && targetWord.charAt(j) == letters) {
            colors[i] = Color.web("#c9b458"); // Gelb für richtige Buchstaben an falscher Stelle
            usedInTargetWord[j] = true;
            lettersFound = true;
            break;
        }
    }

    if (!lettersFound) {
        colors[i] = Color.web("#787c7e"); // Grau für nicht vorkommende Buchstaben
    }
}

Warum zwei Durchgänge?

Durch die Trennung in zwei Phasen wird verhindert, dass ein Buchstabe fälschlicherweise als "gelb" markiert wird, obwohl er an anderer Stelle bereits korrekt (grün) steht. Das usedInTargetWord Array sorgt dafür, dass jeder Buchstabe des Zielworts nur einmal zugeordnet wird.

Rückgabe des Ergebnisses

Farben-Array zurückgeben

return colors;

4 Beispieldurchlauf: "TEICH" und "FISCH"

Um den Algorithmus besser zu verstehen, analysieren wir einen konkreten Fall Schritt für Schritt:

Eingabe: TEICH
Zielwort: FISCH

Schritt 1: Vorbereitung

Initialisierung der Variablen

input = "TEICH";
targetWord = "FISCH";
wordLength = 5;
colors = [null, null, null, null, null];
usedInTargetWord = [false, false, false, false, false];

Schritt 2: Erster Durchgang - Exakte Übereinstimmungen

Position 0: 'T' und 'F'

T ≠ F → Keine Übereinstimmung

colors[0] = #787c7e (grau)

Position 1: 'E' und 'I'

E ≠ I → Keine Übereinstimmung

colors[0] = #787c7e (grau)

Position 2: 'I' und 'S'

I ≠ S → Keine Übereinstimmung

colors[0] = #787c7e (grau)

Position 3: 'C' und 'C'

C = C → Exakte Übereinstimmung

colors[3] = #6aaa64 (grün), usedInTargetWord[3] = true

Position 4: 'H' und 'H'

H = H → Exakte Übereinstimmung

colors[4] = #6aaa64 (grün), usedInTargetWord[4] = true

Stand nach erstem Durchgang:

colors = [grau, grau, grau, grün, grün]
usedInTargetWord = [false, false, false, true, true]

Schritt 3: Zweiter Durchgang - Buchstaben an falscher Position

Position 0: 'T' prüfen

Ist noch nicht grün → Suche 'T' in "FISCH"

Position 0: 'F' ≠ 'T'
Position 1: 'I' ≠ 'T'
Position 2: 'S' ≠ 'T'
Position 3: 'C' ≠ 'T'
Position 4: 'H' ≠ 'T'

Ergebnis: 'T' kommt in "FISCH" nicht vor

colors[0] bleibt #787c7e (grau)

Position 1: 'E' prüfen

Ist noch nicht grün → Suche 'T' in "FISCH"

Position 0: 'F' ≠ 'E'
Position 1: 'I' ≠ 'E'
Position 2: 'S' ≠ 'E'
Position 3: 'C' ≠ 'E'
Position 4: 'H' ≠ 'E'

Ergebnis: 'E' kommt in "FISCH" nicht vor

colors[1] bleibt #787c7e (grau)

Position 2: 'I' prüfen

Ist noch nicht grün → Suche 'I' in "FISCH"

Position 0: 'F' ≠ 'I'
Position 1: 'I' = 'I' → Buchstabe kommt an Position 1 vor
Position 2: 'S' ≠ 'I'
Position 3: 'C' ≠ 'I'
Position 4: 'H' ≠ 'I'

Ergebnis: 'I' kommt in "FISCH" vor

colors[2] wird zu #c9b458 (gelb)

Positionen 3-4: Bereits grün

Diese Positionen werden übersprungen, da sie bereits als korrekt markiert sind.

Endergebnis

Interpretation für den Spieler:
  • T: Kommt im Zielwort nicht vor
  • E: Kommt im Zielwort nicht vor
  • I: Falsch an Position
  • C: Richtig an Position 3
  • H: Richtig an Position 4
